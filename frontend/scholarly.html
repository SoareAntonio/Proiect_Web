1. Introducere

ZooManager este o aplicație Web destinată gestionării informațiilor despre animalele existente într-o grădină zoologică. Aplicația permite vizualizarea unui catalog public de animale, filtrarea acestora după criterii biologice și geografice, autentificarea utilizatorilor și administrarea datelor printr-un panou dedicat administratorilor.

Tehnologii utilizate

  • HTML5, CSS3 și JavaScript pentru interfața utilizatorului;
  • PHP pentru backend și servicii API;
  • Oracle Database pentru stocarea datelor;
  • JSON și XML pentru importul și exportul datelor;
  • JWT pentru autentificare pe bază de token;
  • GitHub pentru versionare și colaborare.

2. Scopul aplicației

Scopul aplicației este de a oferi o platformă simplă și intuitivă pentru consultarea, administrarea, importul și exportul datelor despre animale. Sistemul separă funcționalitățile disponibile utilizatorilor obișnuiți de cele disponibile administratorilor.

3. Categorii de utilizatori

3.1 Utilizator obișnuit

Utilizatorul obișnuit poate accesa catalogul de animale, poate folosi filtrele disponibile, se poate înregistra, se poate autentifica, poate accesa pagina de profil și se poate deloga.

3.2 Administrator

Administratorul are acces la panoul de administrare, unde poate adăuga animale, șterge animale individual, șterge toate animalele, importa date din fișiere externe și exporta datele în formate deschise.

4. Cerințe funcționale principale

4.1 Vizualizarea catalogului de animale

Aplicația trebuie să afișeze animalele existente în baza de date, împreună cu informații precum denumirea populară, denumirea științifică, clasa, originea, regimul alimentar, clima, descrierea și imaginea asociată.

4.2 Filtrarea animalelor

Utilizatorul trebuie să poată filtra animalele după clasă, origine, regim alimentar, climă, existența blănii, capacitatea de a fi dresat și nivelul de periculozitate. Rezultatele trebuie actualizate prin cereri asincrone, fără reîncărcarea completă a paginii.

4.3 Înregistrarea utilizatorilor

Aplicația trebuie să permită crearea unui cont nou prin completarea unui formular cu username, email, parolă și confirmarea parolei. Parola este stocată în baza de date sub formă de hash.

4.4 Autentificarea utilizatorilor

Sistemul trebuie să permită autentificarea utilizatorilor și administratorilor. După autentificare, aplicația salvează un token local și modifică bara de navigare, afișând opțiunile Profil și Logout.

4.5 Pagina de profil

Utilizatorii autentificați trebuie să poată accesa o pagină de profil. Dacă un utilizator neautentificat încearcă să acceseze această pagină, acesta este redirecționat către pagina de login.

4.6 Panoul de administrare

Administratorul trebuie să poată vizualiza animalele într-un tabel, să adauge animale noi, să șteargă animale individual, să șteargă toate animalele și să folosească funcționalitățile de import/export.

4.7 Importul și exportul datelor

Aplicația trebuie să permită importul datelor din fișiere JSON sau XML și exportul datelor existente în formate deschise, precum JSON și XML.

5. Cerințe privind interacțiunea cu utilizatorul

Interfața trebuie să fie clară și intuitivă. Utilizatorul trebuie să poată naviga ușor între catalog, login, register, profil și panoul de administrare. Acțiunile importante, precum ștergerea animalelor, trebuie confirmate înainte de executare.

După autentificare, bara de navigare trebuie să se actualizeze automat. În locul butonului Login trebuie să apară opțiunile Profil și Logout.

6. Cerințe non-funcționale

6.1 Securitate

Aplicația folosește parole hash-uite prin funcții PHP dedicate și autentificare pe bază de token. Operațiile administrative verifică existența unui token valid înainte de executare.

Pentru prevenirea SQL Injection, interogările către baza de date folosesc variabile legate prin oci_bind_by_name. Pentru prevenirea XSS, datele afișate în pagină sunt introduse folosind textContent sau sunt escapate înainte de folosirea innerHTML.

6.2 Performanță

Lista animalelor și filtrele sunt actualizate prin cereri asincrone către backend, fără reîncărcarea completă a paginii.

6.3 Mentenabilitate

Backend-ul este organizat pe principiile MVC. Modelele gestionează accesul la baza de date, controllerele procesează cererile, iar view-urile returnează răspunsuri JSON.

6.4 Portabilitate

Aplicația poate fi rulată local folosind XAMPP, PHP și Oracle Database. Datele de configurare locale sunt separate într-un fișier config.php, care nu este inclus în repository.

7. Arhitectura aplicației

Aplicația folosește o arhitectură în care frontend-ul comunică asincron cu backend-ul prin servicii API. Backend-ul procesează cererile prin controllere, accesează baza de date prin modele și returnează răspunsuri JSON.

8. Criterii de acceptare

  • Catalogul public afișează animalele din baza de date.
  • Filtrele returnează rezultate corecte.
  • Register, login, logout și profil funcționează corect.
  • Administratorul poate adăuga și șterge animale.
  • Ștergerea tuturor animalelor respectă relațiile dintre tabele.
  • Importul și exportul JSON/XML funcționează.
  • Aplicația este protejată împotriva SQL Injection și XSS la nivelul funcționalităților implementate.
  • Configurațiile locale nu sunt expuse în repository.

9. Concluzii

ZooManager oferă o soluție Web pentru gestionarea informațiilor despre animalele unei grădini zoologice, integrând funcționalități de catalog public, autentificare, administrare, import/export și protecții de securitate de bază.
